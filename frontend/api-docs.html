API Reference

FraudShield exposes a REST API that runs every request through a 3-layer detection pipeline — deterministic rules, XGBoost ML scoring, and a live SSE dashboard for human review. Each endpoint returns a fraud score, a final action, and all rule flags that fired.

Base URL http://localhost:8000
Decision Logic

Every response contains one of three actions. The rule engine always runs first — if it BLOCKs, the ML score cannot override it. The ML score can escalate a REVIEW or ALLOW up to BLOCK if the probability exceeds 0.85.

BLOCK
Rule engine fired a block-level flag, or ML score ≥ 0.85
REVIEW
Rule engine flagged a warning, or ML score ≥ 0.55
ALLOW
No rule violations and ML score < 0.55
Response Format

All endpoints return the same JSON shape:

JSON Response
{
  "module":      "payment_fraud",
  "fraud_score": 0.9412,          // 0.0 (safe) → 1.0 (fraud)
  "action":      "block",          // "allow" | "review" | "block"
  "rule_flags": [                   // all rules that fired
    "COUNTRY_MISMATCH: billing=US, shipping=NG",
    "CARD_VELOCITY: 5 different cards used in 24h",
    "DIGITAL_GOODS_RISK: high-value digital purchase on new account"
  ],
  "request_id": "a3f9b2c1"         // short unique ID for audit trail
}
POST /check/login

Checks a login attempt for account takeover signals. Detects brute-force attacks, impossible travel, new device + foreign country combinations, and logins during suspicious hours.

Request Body
FieldTypeRequiredDescription
user_idstringreqInternal user identifier
emailemailreqUser's email address
ip_addressstringreqClient IP of the login request
user_agentstringreqBrowser / device User-Agent header
login_timestampstringreqISO-8601 datetime of the login attempt
last_known_ipstringoptPrevious IP from user's profile
last_known_countrystringoptPrevious country code (e.g. "US")
current_countrystringoptCurrent IP country code
failed_attemptsintegeroptFailed attempts in last 24h (default: 0)
new_devicebooleanoptTrue if device ID not seen before (default: false)
Example Request
Python 
requests.post("http://localhost:8000/check/login", json={
    "user_id":            "u_001",
    "email":              "alice@example.com",
    "ip_address":         "203.0.113.42",
    "user_agent":         "Mozilla/5.0 (Windows NT 10.0)",
    "login_timestamp":    "2025-03-15T03:22:00",
    "last_known_country": "US",
    "current_country":    "RU",
    "failed_attempts":    9,
    "new_device":         True,
})

POST /check/payment

Analyzes a checkout transaction for payment fraud — carding attacks, card velocity, country mismatches, and digital goods risk on new accounts.

Request Body
FieldTypeRequiredDescription
user_idstringreqUser placing the order
order_idstringreqUnique order identifier
amount_usdfloatreqTransaction amount in USD
ip_addressstringreqClient IP at checkout
card_binstringreqFirst 6 digits of card number
billing_countrystringreq2-letter country code (e.g. "US")
shipping_countrystringreq2-letter country code
emailemailreqBuyer email
cards_used_24hintegeroptDistinct cards used in last 24h (default: 1)
orders_24hintegeroptOrders placed in last 24h (default: 1)
account_age_daysintegeroptDays since account creation (default: 0)
is_digital_goodsbooleanoptTrue for software, gift cards, downloads (default: false)
address_matchbooleanoptBilling matches shipping address (default: true)
Example — High-Risk Carding Pattern
Python 
requests.post("http://localhost:8000/check/payment", json={
    "user_id":          "u_042",
    "order_id":         "ord_9981",
    "amount_usd":       1499.99,
    "ip_address":       "185.220.101.5",
    "card_bin":         "411111",
    "billing_country":  "US",
    "shipping_country": "NG",
    "email":            "buyer@example.com",
    "cards_used_24h":   5,
    "orders_24h":       3,
    "account_age_days": 1,
    "is_digital_goods": False,
    "address_match":    False,
})
# → action: "block", fraud_score: 0.94

GET /dashboard/stream

Server-Sent Events endpoint. Connect here to receive a real-time stream of every BLOCK and REVIEW event as they're detected. Sends a ping every 15 seconds to keep the connection alive.

JavaScript Client Example
JavaScript 
const es = new EventSource("http://localhost:8000/dashboard/stream");

es.onmessage = (event) => {
  const data = JSON.parse(event.data);

  if (data.event_type === "ping") return;  // keep-alive

  console.log(data);
  // {
  //   event_type: "fraud_alert",
  //   module: "payment_fraud",
  //   action: "block",
  //   fraud_score: 0.94,
  //   rule_flags: [...],
  //   summary: "Payment $1499.99 ...",
  //   timestamp: "2025-03-15T03:22:00Z"
  // }
};
