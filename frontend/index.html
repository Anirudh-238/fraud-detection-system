How it works

Three layers of protection, working in concert.

Every request passes through all three layers in sequence. Rules catch the obvious. ML catches the subtle. Humans review the edge cases.

01
Rule Engine
Deterministic checks that run in microseconds. Hard-coded thresholds for brute-force, impossible travel, disposable emails, IP velocity, and card abuse. Always consistent — the same input always produces the same decision.
Deterministic
02
XGBoost ML
Four independent gradient-boosted tree models — one per fraud module — trained on labeled synthetic data with realistic fraud ratios. Outputs a probability score between 0.0 and 1.0. Catches patterns rules can't express.
Probabilistic
03
Human-in-the-Loop
Live SSE dashboard streams every BLOCK and REVIEW event to fraud analysts in real time. Borderline cases get a human verdict. Full audit trail in SQLite. The system that real payment companies use.
Real-Time SSE

Developer API

Integrate in minutes.

Four REST endpoints. JSON in, decision out. Each response includes a fraud score, action (BLOCK / REVIEW / ALLOW), and all rule flags that fired.

POST /check/login POST /check/signup POST /check/payment POST /check/promo GET /dashboard/stream
example.py 
import requests

# Check a payment for fraud
response = requests.post(
    "http://localhost:8000/check/payment",
    json={
        "user_id":          "u_042",
        "order_id":         "ord_9981",
        "amount_usd":       1499.99,
        "billing_country":  "US",
        "shipping_country": "NG",
        "cards_used_24h":   5,
        "account_age_days": 1,
        "is_digital_goods": True,
        # ... other fields
    }
)

result = response.json()
# result = {
#   "action": "block",
#   "fraud_score": 0.94,
#   "rule_flags": [
#     "COUNTRY_MISMATCH: US → NG",
#     "CARD_VELOCITY: 5 cards/24h",
#     "DIGITAL_GOODS_RISK: new acct"
#   ]
# }

if result["action"] == "block":
    decline_transaction()
